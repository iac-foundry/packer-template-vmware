# Contributing

Thank you for contributing. Please follow these guidelines:

1. Open an issue to discuss non-trivial changes before sending a PR.
2. Fork the repo and create a topic branch for your work.
3. Keep commits small and focused, and use descriptive commit messages.
4. Include tests or a short verification procedure for changes where applicable.
5. Sign your commits if your org requires signed commits.
6. Add yourself to `CODEOWNERS` if you become a maintainer (ask existing maintainers first).

Pull Request checklist:
- Link to the issue (if any)
- Describe the change and why it's needed
- Run any linting / validation described in `README.md`
